Pseudocode is the use of less technical words or plain understandable language to describe how an algorithm should work. It is often used as a planning and communication tool before writing actual code. Pseudocode has no syntax, but has some rules;

  • capitalise key commands
  • write one statement per line
  • use indentation and spacing

Comments

In programming, a comment is a piece of text that is added to the source code of a program to provide information, explanations, or notes to other developers or to oneself. Comments are not executed or interpreted by the computer and do not affect the behavior or functionality of the program.


In JavaScript, you can use two types of comments: single-line comments and multi-line comments.


Single-line comments start with // and continue until the end of the line. They are used to add explanatory or descriptive text to specific lines of code. Here's an example:

            
// This is a single-line comment
var number = 10; // Assigning the value 10 to the variable 'number'

Multi-line comments start with /* and end with */. They can span multiple lines and are often used to provide longer explanations or to temporarily disable blocks of code. Here's an example:

            
/*
This is a multi-line comment.
It can span multiple lines.
*/
var result = add(5, 3); /* Function call to add() */
